# 一船暴富 (Ship Happens)

《一船暴富》是一款基于浏览器的单机休闲模拟经营与轻肉鸽（Roguelite）航海冒险游戏。玩家在港口用初始资金买船、招募船员、采购资源，然后横渡危险的大海，通过贸易、打劫和处理随机事件来积累财富。

---

## 🛠️ 技术路线 (Technical Architecture)

本项目采用轻量级的纯前端技术栈，力求开发快速、体积小、部署方便。

*   **核心框架**: React + Vite
*   **开发语言**: TypeScript
*   **样式方案**: CSS Modules (纯净、无冲突的组件化样式)
*   **状态管理**: React 原生状态 (`useState` + `useEffect`)，结合纯函数抽取状态计算逻辑，保证游戏状态可预测和可追溯。
*   **数据持久化**: 浏览器原生 `localStorage`。每次状态变更后自动序列化保存，支持断点续玩和重置。
*   **测试框架**: Vitest。专注于测试核心游戏纯函数逻辑（如伤害计算、收益计算、航行推进），确保状态变更的稳定性。
*   **数据驱动**: 游戏所有的道具、船只、敌人和事件均通过独立的静态数据配置 (`data.ts` 和 `events.ts`) 进行驱动，极大方便了后续的内容扩展。

## 📅 开发计划 (Development Roadmap)

### Phase 1: MVP 核心循环构建 (✅ 已完成)
*   搭建 Vite + React + TS 基础工程。
*   实现基础游戏循环：港口准备 -> 航线选择 -> 海上冒险（航行推进、随机事件、简易回合制战斗）-> 结算（到港/返航/沉船）。
*   实装基础数据库（4艘船、7名船员、3种炮弹、4种护甲、6种货物、5种敌人、10种随机事件）。
*   实现本地数据持久化保存和游戏重置功能。

### Phase 2: 内容扩展与机制深化 (🏃 进行中)
*   **事件池扩充**: 将随机事件增加至 30+，引入多步连环事件。
*   **动态市场系统**: 实现基于供需关系的港口货物价格动态波动。
*   **船员技能进阶**: 加入船员等级系统，船员在多次成功航行后可以获得经验并升级技能。
*   **二手船只交易**: 允许玩家将旧船折价出售，降低购买新船的沉没成本。
*   **黑市与势力声望系统**: 根据玩家的海盗行为或行商行为，动态解锁或封锁特定的港口选项。

### Phase 3: 视听体验升级 (⏳ 计划中)
*   **美术素材替换**: 移除纯文本和 Emoji 占位，引入高质量的像素风或 2D 手绘风格的航海资产。
*   **UI 微动效**: 为战斗、伤害结算、获得金币等交互加入反馈动效 (CSS / Framer Motion)。
*   **音效与音乐**: 增加环境音（海浪、海鸥）、UI 交互音效和战斗音效（炮击、木材断裂）。

### Phase 4: 后期玩法 (⏳ 计划中)
*   **无尽模式**: 在通关核心剧情/目标后，解锁随机生成的高难度深海区域。
*   **舰队管理**: 允许玩家拥有多艘船只并组建商队/舰队。
*   **成就系统与元进度**: 跨越不同存档的全局升级或成就解锁，增加重玩价值。

## ⚖️ 数值计划 (Numerical Balancing Plan)

为了保证游戏在“休闲”和“刺激”之间找到完美的平衡，目前的数值设计和未来的调优计划如下：

### 1. 经济系统设计
*   **初始破产保护**: 玩家初始 1000 金币。若完全破产（无船且金币 < 300），提供“商会救济”（300金币），保证最基础的“小渔船”能被购买，防止死档。
*   **收益模型**: 
    *   **低风险贸易**: 近海航线基础收益率约 1.2 - 1.5 倍，稳定但发家慢。
    *   **高风险打劫与走私**: 黑潮/传说航线收益倍率可达 2.0 - 3.5 倍，加上战斗战利品，一次成功出海即可暴富。
*   **资金消耗池**: 维修费是最大的经常性支出。船只越高级，单点耐久的维修费越高（如小渔船 2金币/点，重甲船 8金币/点）。这强制玩家在后期必须选择高收益航线，否则无法覆盖高级船的维修成本。

### 2. 风险与惩罚机制
*   **沉船全损**: 核心肉鸽机制。沉船后将丢失船只、船员、货物、炮弹等一切本次出海资源，仅保留账户现金和声望。这要求玩家在“继续贪一波”和“残血返航”之间做艰难抉择。
*   **返航摩擦成本**: 提前返航将面临货物低价处理（8折）和合同违约金惩罚，但能保住性命和船只。
*   **通缉值 (Bounty)**: 过度打劫商船或攻击巡逻队将提升通缉值。通缉值越高，高危事件（被巡逻队追捕、被同行黑吃黑）的触发概率随之增加，且港口维修可能面临罚款溢价。

### 3. 后期数值平衡方向
*   **动态难度补正**: 如果玩家连续成功出海，系统将微调高危事件出现概率；反之，若玩家连续沉船，将暗中提高“漂浮货箱”、“受损商船（求助）”等正面事件的出现概率。
*   **护甲与伤害缩放**: 增加护甲类型，引入“固定减伤”与“百分比减伤”的差异化防具，以应对后期海怪极高的基础伤害。
*   **货物舱位博弈**: 后期引入“需要占用多舱位”的巨型珍宝，强迫玩家在携带弹药/补给和携带高价货物之间进行极致的舱位博弈。
